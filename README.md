# Essayist Bot

Telegram-бот [@essayist_bot](https://t.me/essayist_bot), который превращает виральный твит
(или заданную вручную тему) в заземлённый русскоязычный авторский разбор и публикует его в
канал владельца — **после ручного одобрения человеком** (human-in-the-loop, без автопостинга).

**Надстройка над [Twidgest](https://github.com/kelbic/twidgest-bot), отдельный процесс.**
Кандидаты-твиты и владение каналами читает из базы Twidgest только на чтение; свой доступ
(Essayist Pro), черновики и расписание хранит в собственной SQLite. Публикует своим
Telegram-токеном.

## Зачем так

Разбор пишется автоматически (план → веб-ресёрч → синтез фактов → черновик → критик-фактчекер),
но финальное решение всегда за человеком — это защищает канал от фактических ошибок модели.
Если веб-поиск не дал ни одного результата, разбор не выпускается (защита от выдумок).

## Мультитенант и доступ

Бот рассчитан на нескольких владельцев каналов (вариант B — отдельный процесс, изолированный
от Twidgest). Три уровня доступа:

1. **Суперадмин** (`ADMIN_USER_ID`) — видит все каналы, выдаёт доступ, сроком не ограничен.
2. **Допуск Essayist Pro** — выдаётся суперадмином, с пробным сроком. Хранится в своей базе
   (`essayist_users`).
3. **Владение каналом** — читается из базы Twidgest (канал принадлежит тому, кто его там завёл).

Связка по Telegram user id: один аккаунт пишет обоим ботам. Пользователь работает только со
своими каналами; суперадмин — со всеми.

### Триал и истечение

`/grant` по умолчанию даёт доступ на 7 дней (`expires_at`). По истечении доступ гаснет
автоматически при следующем действии (ленивая проверка, без фоновых задач). Пустой
`expires_at` = бессрочно (суперадмин, будущие платные). Оплаты пока нет — продление вручную
через `/grant`. Поле `expires_at` готово под будущий биллинг: оплата будет просто двигать дату.

## Команды

**Пользователь:**

- `/start` — регистрация (нужна, чтобы бот мог писать первым — Telegram иначе запрещает).
- `/essay <channel_id>` — топ свежих тем канала (опубликованные скрыты), выбрать кнопкой →
  бот соберёт разбор и пришлёт карточку.
- `/essay <channel_id> all` — полный топ, включая уже опубликованные.
- `/essay <channel_id> <своя тема или ссылка>` — разбор по произвольному тексту.
- `/timer` — панель автоподбора: вкл/выкл и частота по каждому каналу.
- `/help` — справка.

Кнопки карточки: **✅ Опубликовать**, **✍️ Сменить угол** (перегенерация),
**❌ Отклонить** (с причиной: факт-ошибка / стиль / не интересно / другое).

**Суперадмин:**

- `/grant <user_id> [дни|forever] [заметка]` — выдать доступ (по умолчанию 7 дней; `forever` — бессрочно).
- `/revoke <user_id>` — снять доступ.
- `/grants` — список допусков со сроками.

## Автоподбор по таймеру

Внутренний тик — каждые 30 минут. Для каждого канала с включённым автоподбором бот проверяет,
пора ли (по индивидуальной частоте — 3/6/9/12/24 ч, по умолчанию 12), и если да:

- берёт свежую (не старше 24 ч) топовую тему из очереди Twidgest;
- пробует до 3 кандидатов: непоисковые темы (мемы без фактуры → 0 веб-поисков) помечает и
  пропускает, транзиентные сбои API ретраит позже;
- шлёт карточку **владельцу канала** на одобрение (сам не публикует).

Автоподбор по умолчанию выключен (opt-in). Окно свежести 24 ч действует только в автоподборе;
ручной `/essay` показывает темы любого возраста.

### Онбординг канала

Перед включением автоподбора бот через `getChat` проверяет, что добавлен **админом канала с
правом Post Messages** — иначе включить нельзя. Если владелец не нажал `/start`, бот не сможет
ему написать; это видно в `/timer` (отметка ошибки) и в `/grants`.

## Установка

Зависимости (можно переиспользовать venv Twidgest — там они уже есть):

```
pip install aiogram aiohttp aiosqlite python-dotenv
```

Конфиг — скопируй `.env.example` в `.env` и заполни:

```
cp .env.example .env
# затем впиши реальные значения; .env в git не попадает (см. .gitignore)
```

Запуск вручную (для отладки):

```
python3 bot.py
```

Боевой запуск — systemd-сервис (автозапуск + рестарт при падении):

```
# /etc/systemd/system/essayist-bot.service
[Unit]
Description=Essayist HIL bot
After=network-online.target
Wants=network-online.target
StartLimitIntervalSec=300
StartLimitBurst=5

[Service]
Type=simple
WorkingDirectory=/root/essayist-bot
EnvironmentFile=/root/essayist-bot/.env
ExecStart=/root/twidgest-bot/venv/bin/python /root/essayist-bot/bot.py
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```
systemctl daemon-reload && systemctl enable --now essayist-bot
```

Чтобы публикация работала, бот должен быть **админом канала с правом Post Messages**.

## Стоимость

Разбор обходится в центы, не доллары: модели разнесены (дешёвая Haiku на планирование и поиск,
Sonnet на синтез и черновик), плюс нативный веб-поиск Anthropic. Ориентир — единицы долларов в
месяц при десятках разборов.

## Файлы

| Файл             | Назначение                                                              |
|------------------|-------------------------------------------------------------------------|
| `bot.py`         | aiogram-приложение: команды, кнопки, таймер автоподбора                  |
| `auth.py`        | слой доступа: суперадмин / допуск Essayist Pro / владение каналом        |
| `essayist.py`    | движок генерации: пайплайн через Anthropic API                          |
| `candidates.py`  | чтение кандидатов и каналов из базы Twidgest (только на чтение)          |
| `store.py`       | своя SQLite: черновики, доступы (триал), per-channel расписание, реестр /start, пропущенные темы |
| `.env`           | секреты и конфиг (НЕ в git)                                              |

Детали устройства — в [ARCHITECTURE.md](ARCHITECTURE.md).
