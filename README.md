# Essayist Bot

Telegram-бот, который превращает виральный твит (или заданную вручную тему) в
заземлённый русскоязычный авторский разбор и публикует его в канал-витрину —
**после ручного одобрения человеком** (human-in-the-loop, без автопостинга).

Это **отдельный процесс**, не часть Twidgest. Базу Twidgest он читает только на
чтение (берёт кандидаты-твиты и метаданные каналов), свои данные хранит в
собственной SQLite. Публикует своим Telegram-токеном.

## Зачем так

Разбор пишется автоматически (план → веб-ресёрч → синтез фактов → черновик →
критик-фактчекер), но финальное решение всегда за человеком. Это защищает
канал-витрину от фактических ошибок, которые модель может допустить, и при этом
снимает рутину: темы можно выбирать вручную или получать автоподбором по таймеру.

## Возможности

- `/essay <channel_id>` — показать топ-5 свежих тем канала (уже опубликованные
  скрыты), выбрать одну кнопкой → бот соберёт разбор и пришлёт карточку с кнопками.
- `/essay <channel_id> all` — показать полный топ, включая уже опубликованные.
- `/essay <channel_id> <своя тема или ссылка>` — разбор по произвольному тексту.
- Автоподбор по таймеру (по умолчанию раз в 6 ч): бот сам берёт свежую топ-тему
  по каждому активному каналу и шлёт карточку на одобрение.
- `/timer` — панель: включить/выключить каналы для автоподбора, сменить интервал.
- Кнопки карточки: **✅ Опубликовать**, **✍️ Сменить угол** (перегенерация),
  **❌ Отклонить** (с причиной: факт-ошибка / стиль / не интересно / другое).

## Установка

Зависимости (можно переиспользовать venv Twidgest — там они уже есть):

```
pip install aiogram aiohttp aiosqlite python-dotenv
```

Конфиг — скопируй `.env.example` в `.env` и заполни:

```
cp .env.example .env
# затем впиши реальные значения; .env в git не попадает (см. .gitignore)
```

Запуск вручную (для отладки):

```
python3 bot.py
```

Боевой запуск — systemd-сервис (автозапуск + рестарт при падении):

```
# /etc/systemd/system/essayist-bot.service
[Unit]
Description=Essayist HIL bot
After=network-online.target
Wants=network-online.target
StartLimitIntervalSec=300
StartLimitBurst=5

[Service]
Type=simple
WorkingDirectory=/root/essayist-bot
EnvironmentFile=/root/essayist-bot/.env
ExecStart=/root/twidgest-bot/venv/bin/python /root/essayist-bot/bot.py
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```
systemctl daemon-reload && systemctl enable --now essayist-bot
```

Чтобы кнопка «Опубликовать» работала, бот должен быть **админом канала-витрины с
правом Post Messages**.

## Стоимость

Разбор обходится в центы, не доллары: модели разнесены (дешёвая Haiku на
планирование и поиск, Sonnet на синтез и черновик), плюс нативный веб-поиск
Anthropic. Ориентир — единицы долларов в месяц при десятках разборов.

## Файлы

| Файл             | Назначение                                                        |
|------------------|-------------------------------------------------------------------|
| `bot.py`         | aiogram-приложение: команды, кнопки, таймер                       |

| `candidates.py`  | чтение кандидатов из базы Twidgest (только на чтение)             |
| `store.py`       | своя SQLite: черновики, статусы, настройки, дедуп                 |
| `.env`           | секреты и конфиг (НЕ в git)                                        |
